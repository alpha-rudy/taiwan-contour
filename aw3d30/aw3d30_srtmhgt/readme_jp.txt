-----------------------------------------------------
■AW3D30 Geotiff - SRTM HGT 変換用 サンプル スクリプト
-----------------------------------------------------

(a)動作環境
変換用スクリプトはLinuxで動作します。gdal,bash,awkが必要です。
なお、Linux以外には、下記の環境で動作を確認いたしました。

 Windows10 +  Cygwin 2.8.2 ( https://www.cygwin.com/ )
 Windows10 +  OSgeo4W ( https://trac.osgeo.org/osgeo4w/ )


(b)ファイルリスト

  aw3d30_srtmhgt.zip
     aw3d2srtm.sh : 変換用スクリプト

     input  : 入力用フォルダ(AW3D30)
              ※サンプルファイルが入っています。

     output : 出力用フォルダ (SRTM HGT)
              ※サンプルファイルが入っています。


(c)使い方
 1. inputフォルダにAW3D30のGeotifファイルを格納する。 

 2. 変換スクリプトを実行する。
     > bash aw3d2srtm.sh
   
 3. 実行後、outputフォルダ HGTが作成される。
 

(d)変換スクリプト内での処理の流れ
 1. inputフォルダ内の入力ファイルをコンポジットし、
    １つのファイルにする(vrtfile)。
 
 2. SRTM HGTのグリッドへリサンプリングを行い
    同時に１度のタイルに切り出し出す(geotiff)。
 
 3. １度タイルへ切り出したデータ(geotiff)をSRTM HGTへ変換する。


(e)スクリプト使用上の留意点
タイルのエッジ部分のリサンプリングを正確に行うためには、
変換したいタイルだけではなく、その周辺のタイルも必要です。
対象とするエリアよりも、１タイル分、広めにAW3D30データを
ご用意ください。
